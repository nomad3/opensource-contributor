# Contributing to the tracker

Suggestions are welcome when they identify a concrete, public upstream opportunity.

An opportunity should include:

- upstream repository and issue URL;
- why the work is operationally important;
- evidence that the repository is active;
- expected contribution size;
- required hardware or external services;
- known assignee, active PR, or design-decision conflicts;
- the smallest persistent artifact that can be produced.

Do not submit credentials, private logs, employer information, personal contact
details, or copied proprietary code.

Before promoting an item to `active`, confirm that the issue remains open, is not
assigned, has no overlapping pull request, and has sufficiently clear expected
behavior. Only one implementation may be active at a time.
