# Dask #12507 reconnaissance handoff

## Objective

Make optimized Dask array graphs independent of `PYTHONHASHSEED` while
preserving reliable slice pushdown and culling of globally unreachable read
work.

## Completed

- Revalidated [Dask #12507](https://github.com/dask/dask/issues/12507) as open,
  unassigned, and without a matching pull request.
- Inspected current main
  `a54329156cd17a68ef081c83fdc2409ddb423426`.
- Preserved the issue's deterministic-key graph in
  [`reproducers/dask-12507`](../../reproducers/dask-12507).
- Reproduced seven graph sizes from 334 to 728 tasks on Dask 2026.7.1 across
  seeds 0 through 7.
- Confirmed same-seed repeatability in fresh processes.
- Traced the first task-count and declared layer-topology divergence to
  `HighLevelGraph.cull()`, while preserving the caveat that fused task
  representations already differ before culling.
- Confirmed that replacing only the hash-sensitive layer topological traversal
  with a deterministic min-heap makes all eight seeds converge to 438 tasks.
- Kept that deterministic traversal as a causal experiment, not a proposed
  final patch.
- Identified PR #11899 / commit `ef1c013` as the candidate change boundary: it
  removed per-layer output-key filtering before `Layer.cull()`; a parent/commit
  replay is still required before calling it a proven regression boundary.
- Demonstrated directly that two foreign keys make both two-key legacy and
  task-spec layers retain all owned keys through their cardinality fast paths.
- Restored that filtering as a local control and obtained exactly 82 tasks for
  all eight seeds without sorting.
- Encoded every synthetic source block by time, row, and column and confirmed
  seed 0 and seed 7 fixed-path results select the expected blocks in a `float64`
  array with shape `(8, 100, 100)`.

## Risks and unanswered questions

- A deterministic traversal removes cross-seed variation but may merely select
  one arbitrary non-minimal graph; the experiment's 438 tasks exceed the
  smallest observed 334.
- Draft PRs #8560 and #9273 plus open, non-draft, conflicting, dormant PR #10534
  touch adjacent cull, root-fusion, and overlap-materialization design. They do
  not implement hash-seed invariance, but their direction must be reviewed
  before choosing a patch.
- Active PR #12470 refactors the same pre-cull passes for cross-collection use;
  it is not an ownership overlap but its head must be tested before promotion.
- The public real-world 30× xarray case was issue-reported and not available
  locally. Do not extrapolate the toy graph's 2.18× local spread into a proven
  production speedup.
- Canonical layer dependencies are identical before culling, but layer objects
  contain seed-dependent task representations. Do not call the complete
  pre-cull graphs structurally identical.
- Restoring `layer.get_output_keys()` intersection can add work on very large
  graphs. Measure graph-construction time and consider validating ownership in
  the lower-level fast paths as an alternative.
- The issue is only labeled `needs triage`; lack of assignment is not
  maintainer acceptance.
- Dask is cross-science infrastructure rather than an astronomy-only project.
  Its relevance to this lane is the xarray/scientific-pipeline impact.
