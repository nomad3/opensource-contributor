## Decision boundary

This is a source-confirmed compatibility defect in gravitational-wave
preprocessing. The evidence is sufficient to preserve a deterministic future
regression, but not to select a scientific contract or begin implementation.

The current code has two related failure regimes:

1. if the inverse-FFT impulse is shorter than half the requested FIR, a raw
   NumPy broadcasting error escapes;
2. if it is at least half as long but still shorter than the requested FIR,
   GWPy silently returns fewer taps than requested and applies overlapping
   truncation windows.

Maintainers must choose whether an incompatible time-series, ASD, and filter
duration should be rejected, should deliberately shorten the filter, or should
be aligned under a defined ASD extrapolation rule. Those choices have different
spectral and edge-effect consequences. BuildKit PR #6966 also occupies this
portfolio's sole implementation slot.

Decision: **GO for this source-only dossier; WAIT for implementation and
upstream contact.**

## Scientific and operational impact

Whitening weights time-domain strain data by an amplitude spectral density
before transient searches, visualization, and time-frequency analysis. A raw
shape exception is disruptive but visible. The adjacent silent regime is more
important: using fewer coefficients than the requested `fduration` changes the
effective whitening response and the duration of edge contamination without
telling the caller.

Current `TimeSeries.convolve()` preserves the input sample count and does not
crop the corrupted boundaries. It tapers approximately half the effective FIR
at each end. If the series duration is no longer than the intended filter
duration, there is no uncontaminated interior under that intended filter.

This dossier does not establish that any published result is wrong. It
establishes a source-level path by which supplied-ASD whitening can fail
cryptically or silently use a shorter filter than requested.

## Current call path

Pinned current source:

- [`TimeSeries.whiten()`](https://gitlab.com/gwpy/gwpy/-/blob/db3e442238130618f3ad477b21b670ea9fc463cc/gwpy/timeseries/timeseries.py#L2289-2404)
- [`FrequencySeries.interpolate()`](https://gitlab.com/gwpy/gwpy/-/blob/db3e442238130618f3ad477b21b670ea9fc463cc/gwpy/frequencyseries/frequencyseries.py#L316-347)
- [`_truncate_impulse()` and `fir_from_transfer()`](https://gitlab.com/gwpy/gwpy/-/blob/db3e442238130618f3ad477b21b670ea9fc463cc/gwpy/signal/filter_design.py#L1012-1096)
- [`TimeSeries.convolve()`](https://gitlab.com/gwpy/gwpy/-/blob/db3e442238130618f3ad477b21b670ea9fc463cc/gwpy/timeseries/timeseries.py#L2574-2650)

The path is:

1. `whiten()` chooses or receives an ASD.
2. Both branches interpolate that ASD to frequency spacing `1 / duration`.
3. `fir_from_transfer(1 / asd, ntaps)` performs an `irfft()` without an
   explicit output length.
4. `_truncate_impulse()` multiplies the first and last halves of that impulse
   by the two halves of an `ntaps`-long window.
5. `fir_from_transfer()` rolls the result and slices `[0:ntaps]`.
6. `convolve()` uses the actual returned FIR length and emits a series with the
   original sample count and timestamps.

The supplied-ASD path is not missing frequency-resolution interpolation.
Instead, interpolation preserves the supplied ASD's frequency span, which can
produce an impulse too short for the requested filter.

## Exact length algebra

Let:

- `N` be the time-series sample count;
- `fs` be its sample rate;
- `T = N / fs` be its duration;
- `A` be the supplied ASD bin count;
- `df_asd` be the supplied ASD spacing;
- `B = (A - 1) * df_asd` be its represented frequency span;
- `K = int(fduration * fs)` be the requested tap count for a named or tuple
  window;
- `H = floor(K / 2)`.

`FrequencySeries.interpolate(1 / T)` produces

```text
M = int(rint(B * T + 1))
```

bins. Because `np.fft.irfft()` receives no explicit length, the impulse length
is

```text
I = 2 * (M - 1)
  = 2 * (int(rint((A - 1) * df_asd * T + 1)) - 1).
```

For a custom window, `K = len(window)` instead.

The first truncation multiply has operand lengths

```text
min(I, H) and K - H.
```

For a positive, documented even `K >= 2` and a non-empty impulse, it succeeds
exactly when `I >= K / 2`. Therefore:

- `I < K / 2`: raw broadcasting failure;
- `K / 2 <= I < K`: no broadcast failure, but the two half-windows overlap
  and `[0:K]` returns only `I` coefficients;
- `I >= K`: the function can return the requested `K` coefficients.

For full-Nyquist, grid-aligned ASD coverage, `B = fs / 2` and `I` is
approximately `N`. The boundaries then simplify to:

- `T < fduration / 2`: broadcast failure;
- `fduration / 2 <= T < fduration`: silent-shortening regime;
- `T >= fduration`: enough impulse samples for the requested tap count.

Odd `K` is a separate validation gap. For odd `K >= 3`, the left slice can
contain at most `floor(K / 2)` elements while the corresponding window half
contains `ceil(K / 2)`, so the first multiply is intrinsically mismatched.
`K = 1` is a degenerate exception: NumPy permits the empty left slice to
broadcast against the one-element window half, after which the middle-zeroing
path can return a zero one-tap FIR. The FIR documentation requires an even tap
count, but `whiten()` does not validate it.

Rounding in `rint()` and `irfft()` parity can move a boundary by one or two
samples. A future patch should test the exact integer relation, not only the
continuous approximation.

## What the traceback proves

For an even filter, the two reported operand shapes establish:

```text
I = 3072
K / 2 = 4096
K = 8192
```

They do not establish:

- that the time series itself had 3,072 samples;
- a 4,096-Hz sample rate;
- the default two-second `fduration`;
- a particular ASD bin count or spacing.

One exact source-derived reconstruction is:

```text
N = 4096
fs = 4096 Hz
T = 1 second
fduration = 2 seconds
K = 8192
```

A full-Nyquist ASD with 2,049 bins at 1 Hz gives `I = 4096`: it avoids the raw
exception but remains in the silent-shortening regime. A supplied ASD with
1,537 bins at 1 Hz, ending at 1,536 Hz, gives `I = 3072` and reproduces the
reported `(3072,)` versus `(4096,)` mismatch.

This is a deterministic algebra control, not a reconstruction of the
reporter's undisclosed inputs.

## Why `asd=None` can behave differently

An internally generated ASD ordinarily covers zero through the input
Nyquist frequency, yielding `I` approximately equal to `N`. A supplied ASD can
cover a narrower band. Interpolating both to `1 / T` changes their resolution
but retains those different spans, so the supplied branch can cross the
`I < K / 2` boundary while the generated branch does not.

If the generated and supplied ASDs have the same full-Nyquist grid, the
inspected post-ASD source paths converge. The issue omits enough metadata that
the exact reason for its branch contrast remains an informed source-level
inference, not a reproduced fact.

## Contract risks

### Rejecting

Early validation is the smallest behaviorally explicit change. It may reject
inputs that currently pass through the silent-shortening regime, so the
exception type, message, and release-note treatment need maintainer approval.

### Shortening

Clamping `K` to `I` prevents the broadcast but changes the requested
`fduration`, whitening response, normalization, and edge-contamination
duration. It also risks treating an ASD coverage mismatch as if it were merely
a short record. A silent clamp is not an acceptable resolution.

### Aligning

Extending an ASD to Nyquist requires values for frequencies the caller did not
supply. Zero-padding, constant extrapolation, or interpolation beyond the
represented domain each implies a different transfer function. Aligning the
filter upward can also move the failure into convolution when half the FIR is
longer than the input.

### Frequency compatibility

Current source does not explicitly validate supplied-ASD Nyquist coverage,
starting frequency, or compatibility with the time-series sample rate.
Expanding #1850 into a general spectral-grid redesign would exceed this bounded
issue unless maintainers request it.
